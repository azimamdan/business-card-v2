## Components

### Public Profile (`src/app/[username]`)
- **Purpose:** Renders the public-facing canvas for a user.
- **Key Features:** Server-side data fetching for SEO, staggered animation reveals using `AnimatedSection`, loading skeletons.
- **Dependencies:** `getProfile`, `getBlocks`, `AnimatedSection`, `BlockRenderer`.

### Dashboard (`src/app/dashboard`)
- **Purpose:** Primary interface for profile customization and block management.
- **Key Features:** Drag-and-drop reordering, avatar uploading, live preview, mobile-responsive tabs with `AnimatePresence`.
- **Dependencies:** `@dnd-kit`, `ProfileForm`, `AvatarUpload`, `BlockList`.

### Block Renderers (`src/components/blocks`)
- **Purpose:** Modular components that display specific data types (Hero, V-Card, etc.).
- **Key Features:** Responsive design, Framer Motion transitions, `next/image` optimization.
- **Dependencies:** `motion`, `Image`, `Lucide Icons`.

### Server Actions (`src/lib/actions`)
- **Purpose:** Bridge between the Client and Supabase Database.
- **Key Features:** `updateProfile`, `addBlock`, `reorderBlocks`.
- **Dependencies:** `createClient` (Supabase Server).

## Data Flow

1. **Authentication:** User logs in via `src/app/login` -> Supabase Auth -> Middleware protects `/dashboard`.
2. **Profile Creation:** User updates info in Dashboard -> Profile Server Action -> Supabase `profiles` table.
3. **Block Creation/Ordering:** User adds/moves blocks -> Block Server Actions -> Supabase `blocks` table.
4. **Image Upload:** Dashboard -> `AvatarUpload` -> Supabase Storage -> `avatar_url` updated in `profiles`.
5. **Consumption:** Visitor hits `canvas.to/username` -> `[username]/page.tsx` fetches profile/blocks -> `BlockRenderer` maps data to components.
